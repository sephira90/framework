# 2026-03-14 Permission Escalation Rule

## Context
- Потребовалось явно зафиксировать правило поведения при сбоях, связанных с правами доступа и ограничениями среды.
- Цель: не останавливать работу молча и не подменять задачу обходными решениями, если проблема упирается в permissions.

## Key Findings
- Блокеры прав и доступа являются частью реального operational context проекта.
- Если такое ограничение не зафиксировано в правилах, возникает риск молчаливых workaround-решений, которые искажают задачу или скрывают причину сбоя.

## Decisions
- В `AGENTS.md` добавлено правило: при permission-related blocker нужно явно назвать недостающее право, запросить права у пользователя и не обходить проблему молча неэквивалентным способом.
- Дополнительно зафиксировано, что статус запроса прав по возможности должен попадать в execution log.

## What This Changes
- Теперь permission failures считаются формализованным блокером процесса.
- Запрос прав становится обязательной частью рабочего протокола, если без него нельзя честно выполнить задачу.
